# MVP仕様書：Tasting Log App

## 1. コンセプト

飲んだお酒を、写真・名前・場所・テイスティングメモと一緒に記録し、あとで一覧で見返せるシンプルなテイスティングログアプリ。

最初のMVPでは、AI分析・マップ表示・SNS機能などは入れず、まずは「記録する」「一覧で見返す」に集中する。

## 2. MVPの目的

このMVPの目的は、以下を検証すること。

* 飲んだお酒を自分が継続して記録したくなるか
* 記録した内容をあとで見返したくなるか
* 写真・名前・種類・場所・日付だけで一覧として十分に価値があるか
* 種類フィルターが振り返りに役立つか

## 3. 対象ユーザー

主な対象は、お酒を学びながら楽しみたい個人ユーザー。

特に以下のような人を想定する。

* ワイン・日本酒・ビールなどを飲んだ記録を残したい人
* テイスティングメモを自分の言葉で残したい人
* 飲んだ場所や日付と一緒に記録を振り返りたい人
* 専門的すぎる管理アプリではなく、シンプルな記録アプリを使いたい人

## 4. MVPで作る機能

### 4.1 記録作成機能

ユーザーは飲んだお酒の記録を作成できる。

記録する項目は以下。

| 項目          | 必須 | 内容                               |
| ----------- | -- | -------------------------------- |
| 写真          | 任意 | お酒・ボトル・グラスなどの写真                  |
| お酒の名前       | 必須 | 例：Josef Chromy Chardonnay 2022   |
| 種類          | 必須 | ワイン、日本酒、ビール、その他                  |
| 産地・品種       | 任意 | 例：タスマニア / シャルドネ                  |
| 飲んだ場所       | 任意 | 例：Sake Restaurant & Bar Brisbane |
| 自由テイスティングメモ | 任意 | 香り、味、印象、気づきなどを自由に記録              |
| 飲んだ日        | 必須 | デフォルトは当日の日付                      |

### 4.2 一覧表示機能

作成した記録をカード形式で一覧表示する。

カードに表示する項目は以下。

* 写真
* お酒の名前
* 種類
* 飲んだ場所
* 飲んだ日

### 4.3 種類フィルター機能

一覧画面で、お酒の種類ごとに記録を絞り込める。

フィルター項目は以下。

* すべて
* ワイン
* 日本酒
* ビール
* その他

## 5. MVPでは作らない機能

初期MVPでは、以下の機能は作らない。

* AIによるテイスティングメモ整理
* マップ表示
* 詳細な分析機能
* SNS共有
* フォロー機能
* レコメンド機能
* ラベル読み取り機能
* 産地・品種ごとの詳細フィルター
* ユーザー同士のレビュー機能

これらは記録が十分に溜まり、必要性が確認できた後に検討する。

## 6. 画面構成

### 6.1 一覧画面

役割：記録したお酒を見返すメイン画面。

表示内容：

* 画面タイトル
* 種類フィルター
* 記録カード一覧
* 新規記録作成ボタン

カード表示内容：

* 写真
* お酒の名前
* 種類
* 飲んだ場所
* 飲んだ日

### 6.2 新規記録画面

役割：新しいお酒の記録を作成する画面。

入力項目：

* 写真
* お酒の名前
* 種類
* 産地・品種
* 飲んだ場所
* 自由テイスティングメモ
* 飲んだ日

アクション：

* 保存
* キャンセル / 戻る

### 6.3 詳細画面

MVPに含める場合は、一覧カードをタップして詳細を表示する。

表示内容：

* 写真
* お酒の名前
* 種類
* 産地・品種
* 飲んだ場所
* 飲んだ日
* 自由テイスティングメモ

編集機能はMVP初期では任意とする。

## 7. データ設計

### tasting_records

| カラム名           | 型        | 内容          |
| -------------- | -------- | ----------- |
| id             | string   | 記録ID        |
| name           | string   | お酒の名前       |
| type           | string   | 種類          |
| origin_variety | string   | 産地・品種       |
| place          | string   | 飲んだ場所       |
| tasting_note   | text     | 自由テイスティングメモ |
| image_url      | string   | 写真URL       |
| drank_at       | date     | 飲んだ日        |
| created_at     | datetime | 作成日時        |
| updated_at     | datetime | 更新日時        |

## 8. 種類の定義

初期MVPでは、種類は以下に固定する。

```ts
type DrinkType = "wine" | "sake" | "beer" | "other";
```

表示名は以下。

```ts
const drinkTypeLabels = {
  wine: "ワイン",
  sake: "日本酒",
  beer: "ビール",
  other: "その他",
};
```

## 9. 産地・品種の扱い

産地・品種は、MVPでは1つの自由入力フィールドとして扱う。

理由：

* ワイン、日本酒、ビールで必要な項目が異なるため
* 最初から細かく分けると入力が複雑になるため
* MVPでは記録のしやすさを優先するため

例：

* タスマニア / シャルドネ
* 山田錦 / 兵庫
* Marlborough / Sauvignon Blanc
* 不明

## 10. 成功指標

MVPの成功指標は以下。

* 自分が10件以上記録する
* 記録作成が面倒に感じない
* 一覧を見返したときに価値を感じる
* 種類フィルターを使いたくなる
* 次に追加したい機能が自然に見えてくる

## 11. 今後追加を検討する機能

MVP後に検討する機能。

* マップ表示
* AIによるメモ整理
* 品種・産地・国別フィルター
* 評価機能
* タグ機能
* お気に入り機能
* 統計・分析機能
* 写真からのラベル読み取り
* 記録のエクスポート
* SNS共有
